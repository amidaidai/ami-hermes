## A. 置信度公式

置信不再拍脑袋，按以下公式计算后取整（1-5）：

```text
置信原始分 = (评分/13 × 3.0) + (数据等级权重 × 1.0) + (模型匹配度 × 0.5) + (历史胜率加分 × 0.5)

其中：
- 数据等级权重：A=1.0, B=0.7, C=0.3
- 模型匹配度：所有入场前提条件全部满足=1.0，缺1个=0.6，缺2个=0.3，缺≥3个=0
- 历史胜率加分：该模型历史胜率>50%→0.5, 40-50%→0.3, <40%或统计不足→0
  
换算：
原始分 ≥4.0 → 置信 5/5
原始分 3.2-3.9 → 置信 4/5
原始分 2.4-3.1 → 置信 3/5
原始分 1.6-2.3 → 置信 2/5
原始分 <1.6 → 置信 1/5
```

示例：评分9，数据A，模型匹配度0.6，历史胜率0.3
→ 9/13×3.0 + 1.0×1.0 + 0.6×0.5 + 0.3×0.5 = 2.08+1.0+0.3+0.15 = 3.53 → **置信 4/5**

## B. 加权评分器

原13分制7维等权不合理。改为加权：

| 维度 | 原分值 | 加权系数 | 加权满分 | 说明 |
|------|--------|---------|---------|------|
| 结构 | 0-3 | ×1.5 | 4.5 | 技术结构是根，最高权重 |
| 周期 | -2-2 | ×1.0 | 2.0 | 高低周期一致性 |
| 订单流 | 0-2 | ×1.2 | 2.4 | CVD/Taker确认，但受CVD等级影响 |
| 衍生品 | 0-2 | ×0.8 | 1.6 | 辅助确认，非主驱动 |
| 催化 | 0-1 | ×1.0 | 1.0 | 事件驱动权重 |
| 风控 | 0-2 | ×1.5 | 3.0 | 安全边际，与结构同权 |
| 情绪 | -1-1 | ×0.5 | 0.5 | 噪音大，低权重 |

```text
加权总分 = 结构×1.5 + 周期×1.0 + 订单流×1.2 + 衍生品×0.8 + 催化×1.0 + 风控×1.5 + 情绪×0.5
满分 = 4.5+2.0+2.4+1.6+1.0+3.0+0.5 = 15.0

换算为13分制显示：
显示分 = round(加权总分 / 15.0 × 13, 1)
```

**CVD等级降权：** 当CVD为C级时，订单流维度得分自动打6折（0-2 → 0-1.2）。

**判定阈值（不变）：**
```text
加权总分/15.0 ≥ 10/15 → A做多/A做空
加权总分/15.0 6-9.9/15 → B等待
加权总分/15.0 <6/15 → X禁做
（其余禁做规则不变）
```

## C. 量价关系矩阵

成交量必须做相对判断，不再"感觉大/小"：

```text
量比 = 当前K线成交量 / 前20根均量

┌──────────┬────────────────────┬────────────────────┐
│ 量比范围 │ 上涨中             │ 下跌中             │
├──────────┼────────────────────┼────────────────────┤
│ >2.0     │ 爆量拉升→加速/高潮 │ 爆量杀跌→恐慌/接筹  │
│ 1.5-2.0  │ 放量确认→强势      │ 放量确认→派发      │
│ 0.7-1.5  │ 正常→中性          │ 正常→中性          │
│ 0.3-0.7  │ 缩量涨→衰竭警告    │ 缩量跌→衰竭卖出    │
│ <0.3     │ 地量→流动性真空     │ 地量→无人区        │
└──────────┴────────────────────┴────────────────────┘

入场量确认规则：
- 确认K的成交量 ≥ 前3根均量 × 1.5 → 量确认有效（+1 风控分）
- 确认K的成交量 < 前3根均量 × 0.5 → 量确认不足（降仓位一级）
```

输出模板（替换原"量能解读"）：

```text
⑥ 量能：量比 `{ratio}x` · {放量/正常/缩量/地量} · {解读}
```

## D. 模型强制 Checklist

五类模型不再主观判定，使用标准化检查项。

### 通用入口检查（所有模型必过）

```text
□ 1. 数据等级 ≥ B？（C级→禁做A单）
□ 2. 无重大事件前30分钟禁做限制？
□ 3. R:R ≥ 1:2 可构造？
□ 4. 日损未达30U上限？
□ 5. 连续亏损<3笔？
□ 6. 入场价离现价 ≤ 0.5 ATR？（超过→禁追）
□ 7. 高周期（4h/1h）与执行方向不冲突？
  
全部勾选通过 → 进入模型特定检查
任一未勾 → 状态降级（B等待或X禁做）
```

### 模型特定检查（按各模型§5触发清单）

每张卡输出时必须带checklist标记：

```text
模型检查：□通用入口 7/7 ✓ → □VWAP反抽 入场4/4 · 触发3/3 · 确认2/3
```

## E. 数据时效监控

采集器输出snapshot时自动打时间戳。分析卡输出时标注：

```text
数据新鲜度：
- 价格：{n}秒前（Binance API）
- 费率：{n}分钟前
- OI：{n}分钟前
- 宏观（DXY/VIX）：{n}分钟前

时效评级：
- 所有关键数据 ≤5分钟 → 新鲜
- 价格 ≤30秒 + 衍生品 ≤15分钟 → 可用
- 任一类 >15分钟 → 过期警告
- 任一类 >30分钟 → 需刷新
```

数据采集器 `data_gatherer.py` 默认缓存30秒，调用即刷新。

## G. 催化源独立性分级

| 等级 | 条件 | 处理 |
|------|------|------|
| AA | 彭博/路透 + 金十/日经，独立渠道，信息不互引 | 全额计分 |
| A | 金十 + 另一个非聚合源（如官方公告原文），独立 | 全额计分 |
| B | 两个聚合源，信息可能互引但无法确认（如金十+CoinDesk） | 催化分打8折 |
| C | 单源或两个源明确互引 | 催化分打5折 |
| D | 无可靠源或全为二手引用 | 催化0分，不论内容 |

输出模板：

```text
⑤ 催化：{内容摘要} — {AA/A/B/C/D}级 · {单源/双源互引/独立双源} — {影响解读}
```

## I. ATR 计算标准

统一使用 4h ATR（14周期），作为所有止损距离的基准：

```text
ATR 4h(14) ≈ (过去14根4hK线 True Range 均值)
True Range = max(高-低, |高-前收|, |低-前收|)

从采集器直接取或从TV图表拉取。
```

输出规范：

```text
⑤ ATR(4h,14)：`{atr}` — 止损距离 `{stop_dist}` = {stop_dist/atr}x ATR
```

止损距离规则：
- 正常仓：止损 ≥ 0.5 ATR 且 ≤ 1.5 ATR
- 止损 < 0.3 ATR：降仓位至轻仓（太近易扫）
- 止损 > 2.0 ATR：R:R不合格（太远损不起）

## J. CVD 等级处理规则

| CVD等级 | 订单流评分 | 模型确认 | 仓位影响 | 说明 |
|---------|----------|---------|---------|------|
| A级 | 全额(0-2) | 全效 | 正常 | 真实futures aggTrades |
| B级 | 全额(0-2) | 全效 | 正常 | Binance Taker统计 |
| C级 | **×0.6** | **仅半效** | **轻仓** | K线估算/spot proxy |
| 无 | 0 | 不可用 | 禁做A单 | 无订单流数据 |

当前环境：CVD 始终为 C级（TradingView K线估算 或 spot 近似），因此：
- 订单流得分自动打6折
- 使用CVD确认模型时，需多重确认（至少再加1个独立信号）
- 正常仓位的最高上限为半仓
