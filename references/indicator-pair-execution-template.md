## 2. 头部模板

每张分析卡必须先给结论，不许先堆数据。排列逻辑：品种锚定在最前，之后按"框架→参照→结论→逻辑→量化→动作→依据→风险→可信度"层层递进，信息密度递减但重要性不减。

```text
**① 品种：{SYMBOL}**
**② 周期：{4h背景} → {1h结构} → {15m执行} → {5m触发}  [{分析范围}]**
**③ 现价：`{price}` · 24h高 `{high}` · 24h低 `{low}` · 日跌/涨 {n}%**
**④ 状态：{A做多/A做空/B等待/X禁做}**
**⑤ 模型：{VWAP反抽/VAH回收/VAL回收/POC拒绝/扫流动性回收/突破接受/无}**
**⑥ 评分：{总分}/13 → {A/B/X}（{方向判} · {压制原因}）**
　　　{各维打分}
**⑦ 决策：{一句话动作} · 置信 {n}/5**
**⑧ 原因：{核心依据} · 有效期 {expiry}**
**⑨ 仓位：{正常/半仓/轻仓/禁止} · 最大风险 `{risk_usd}U`**
```

扫读路径：品种 → 周期 → 价格 → 能不能做 → 为什么 → 几分 → 做什么 → 凭什么 → 多少钱 → 数据可信吗。

## 3. 状态评分器

| 项目 | 分值 | 评分标准 |
|------|------|----------|
| 结构 | 0-3 | 趋势、价值区、关键位是否清楚 |
| 周期 | -2-2 | 高周期是否支持低周期执行方向 |
| 订单流 | 0-2 | CVD、Taker、吸收、背离是否确认 |
| 衍生品 | 0-2 | Funding、OI、多空、Basis 是否支持 |
| 催化 | 0-1 | 新闻、宏观、日历是否支持 |
| 风控 | 0-2 | R:R、止损距离、日限是否合格 |
| 情绪 | -1-1 | 极端拥挤降权，恐慌只小幅加权 |

判定规则：

```text
10-13分：A做多/A做空
6-9分：B等待
0-5分：X禁做
数据C级：最高 B等待
高周期明显冲突：最高 B等待
4h与1h同时冲突：X禁做
R:R低于1:2：X禁做
日损达到30U：X禁做
连续亏损3笔：X禁做
重大数据前30分钟：X禁做
```

## 4. 固定五类模型

| 模型 | 适用 | 必要条件 | 禁做条件 |
|------|------|----------|----------|
| VWAP反抽 | 趋势延续 | 高周期同向，反抽/回踩 VWAP 后拒绝，CVD/Taker 不冲突 | 重新站回 VWAP 并接受 |
| VAH/VAL回收 | 假突破反转 | 打出价值区外失败后回收，有吸收或背离 | 突破后接受并回踩不破 |
| POC拒绝 | 平衡区拒绝 | 回到 POC 附近失败，适合区间衰竭或趋势中继 | POC 被收复并接受 |
| 扫流动性回收 | 扫损后反转 | 扫前高/前低或 session high-low 后快速收回 | 扫后放量延续，不收回 |
| 突破接受 | 顺势突破 | 突破 VAH/VAL/VWAP 后回踩不破，成交量与 CVD 确认 | 假突破回落价值区内 |

禁止临场发明第六类模型。若不属于五类之一，状态只能是 B等待或 X禁做。


## 5. 模型触发清单

五类模型必须按清单检查，不满足“入场前检查 + 触发条件 + 确认条件”时，只能等待，不能提前进。

### 5.1 VWAP反抽

```text
入场前检查：
① 周期：4h/1h 方向同向，至少不明显冲突
② 位置：价格回踩或反抽 VWAP，不能离 VWAP 超过 0.5ATR
③ 结构：前一段趋势没有明显衰竭，价值区仍支持延续
④ 风控：止损可放在 VWAP 外侧或结构低/高外侧，R:R ≥ 1:2

触发条件：
① 价格触碰 VWAP 后出现拒绝
② 5m/15m 收线不重新接受 VWAP
③ Taker 或 CVD 不与方向冲突

确认条件：
① 顺向K线重新启动
② 反向量能衰竭
③ 回踩/反抽不破确认位

失效条件：
① 重新站回 VWAP 并连续接受
② CVD 与方向持续背离
③ 高周期关键位被反向收复

监控位写法：
① VWAP触碰提醒
② VWAP接受失效提醒
③ 启动确认提醒

复盘标签：VWAP反抽 · 反抽/回踩 · 顺势延续
```

### 5.2 VAH/VAL回收

```text
入场前检查：
① 位置：价格打出 VAH 上方或 VAL 下方
② 结构：突破后未形成接受，出现假突破嫌疑
③ 订单流：扫出流动性后 CVD/Taker 没有继续顺向放大
④ 风控：止损可放在扫出极值外侧，R:R ≥ 1:2

触发条件：
① 价格重新回到价值区内
② 5m/15m 收线确认回收
③ 扫出一侧流动性后无法延续

确认条件：
① 回收后不再跌/涨回价值区外
② CVD 出现背离或主动成交转向
③ 目标至少能看到 POC 或价值区另一侧

失效条件：
① 重新突破 VAH/VAL 并接受
② 回踩价值区边界不破后继续顺势
③ 回收后量能不足且价格横住不动

监控位写法：
① VAH/VAL扫出提醒
② 价值区回收提醒
③ POC磁吸提醒

复盘标签：VAH回收 / VAL回收 · 假突破 · 价值区回归
```

### 5.3 POC拒绝

```text
入场前检查：
① 位置：价格回到 POC 或高成交节点附近
② 背景：当前不是强趋势单边接受状态
③ 价值：POC 附近有明确拒绝空间
④ 风控：止损可放在 POC 接受后的外侧，R:R ≥ 1:2

触发条件：
① 价格触碰 POC 后失败
② 5m/15m 出现拒绝K线或反向主动成交
③ POC 未被连续收复

确认条件：
① 拒绝后离开 POC 区域
② 成交量支持反向移动
③ CVD/Taker 至少不反向冲突

失效条件：
① POC 被收复并接受
② 回踩 POC 不破并继续原方向
③ 价格围绕 POC 反复横盘，方向不清

监控位写法：
① POC触碰提醒
② POC拒绝确认提醒
③ POC接受失效提醒

复盘标签：POC拒绝 · 平衡区拒绝 · 区间/中继
```

### 5.4 扫流动性回收

```text
入场前检查：
① 位置：接近前高、前低、亚洲高低、纽高纽低或明显止损池
② 背景：高周期没有强烈反向压制
③ 订单流：扫出后观察主动成交是否衰竭
④ 风控：止损可放在扫出极值外，R:R ≥ 1:2

触发条件：
① 价格刺破关键高/低
② 刺破后快速回收
③ 5m/15m 收回关键位内侧

确认条件：
① CVD 背离或 Taker 追单失败
② 回收后不再二次跌破/突破扫点
③ 出现反向结构位突破

失效条件：
① 扫后继续放量延续
② 回收失败并在扫点外接受
③ 高周期同步破位

监控位写法：
① 扫前高/前低提醒
② 快速回收提醒
③ 扫后延续失效提醒

复盘标签：扫流动性回收 · 扫损 · trapped flow
```

### 5.5 突破接受

```text
入场前检查：
① 位置：接近 VAH/VAL/VWAP/前高前低等结构位
② 背景：高周期支持突破方向，至少不冲突
③ 量能：突破前不是明显衰竭
④ 风控：回踩确认位到止损位的 R:R ≥ 1:2

触发条件：
① 价格突破关键位
② 5m/15m 收线站稳关键位外侧
③ 回踩关键位不破

确认条件：
① CVD/Taker 顺向确认
② OI 增加但不过度拥挤
③ 回踩后出现二次启动

失效条件：
① 突破后快速回落价值区内
② 回踩关键位跌破/上破失败
③ CVD 背离且 Taker 追单失败

监控位写法：
① 突破提醒
② 回踩接受提醒
③ 假突破回落提醒

复盘标签：突破接受 · 回踩确认 · 顺势突破
```

## 6. 数据可信度

| 等级 | 条件 | 处理 |
|------|------|------|
| A | 价格3源一致，偏差≤0.15%；衍生品可用；催化2源确认；关键数据≤15分钟 | 可正常仓或半仓 |
| B | 价格2源一致，偏差≤0.30%；衍生品部分可用；催化单源或旧于30分钟 | 半仓或轻仓 |
| C | 价格单源或偏差>0.30%；CVD/衍生品缺失；催化无法确认 | 最高 B等待，通常轻仓或禁做 |

必须输出：

```text
数据：{A/B/C} · {来源数量}源一致 — {冲突/一致解读}
```

## 7. 衍生品六件套

每次分析必须检查：

```text
Funding：多空付费方向
OI：杠杆是否增加
多空：账户拥挤方向
Taker：主动买卖强弱
Basis：合约溢价/贴水
清算：上下方磁吸区
```

解释矩阵：

```text
多头拥挤：Funding正 + OI升 + 多空偏多 + 价格不涨 → 防杀多
空头拥挤：Funding负 + OI升 + 多空偏空 + 价格不跌 → 防逼空
逼空风险：Funding负 + OI高 + 站回关键位 + Taker转买 → 空单降仓
杀多风险：Funding正 + OI高 + 跌破关键位 + Taker转卖 → 多单降仓
无效信号：OI降 + Funding中性 + Taker混乱 → 不加分
```

## 10. 仓位计算器

```text
风险金额 = min(计划风险, 风控允许风险, 剩余日限)
止损距离 = abs(入场 - 止损)
数量 = 风险金额 / 止损距离
名义 = 数量 × 入场
保证金 = 名义 / 杠杆
最大亏损 = 风险金额
```

输出必须写：

```text
仓位：风险 `{风险金额}` ÷ 距离 `{止损距离}` = 数量 `{数量}` · 名义 `{名义}` · 保证金 `{保证金}` · 最大亏损 `{最大亏损}` · R:R `{rr}`
```

## 17. 品种模块

### BTC/ETH

- 必看：Funding、OI、多空、Taker、Basis、清算热力图
- 催化：ETF、监管、交易所、宏观流动性、美元、美股风险偏好
- 权重：结构40%，衍生品25%，订单流15%，风控10%，新闻/社区10%

### 山寨

- 必看：BTC相关性、板块强弱、解锁、深度、滑点、做市异常
- 默认半仓以下；数据不到 A，不给 A做多/A做空
- 滑点大、成交额低、盘口薄：X禁做

### 黄金 XAU

- 必看：DXY、美债收益率、Fed预期、CPI/PCE/NFP/FOMC、London/NY时段、地缘突发
- 权重：宏观30%，结构35%，订单流15%，新闻15%，情绪5%

### 外汇/股指

- 必看：DXY、美债、VIX、交易时段、宏观日历、板块轮动

## 18. 格式规范

- 纯纵向结构，层层递进
- 无 `｜`，无方括号，无装饰 emoji，仅等级可用 🟢🔵🟡🔴
- 除固定英文术语外，其余字段中文化
- 价格全部用反引号
- 结构线、方向、操作指令加粗
- 每行数据必须带解读，不裸放数字
- 禁做条件必须明确，不写"谨慎"这种空话
